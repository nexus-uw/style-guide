Input Guidelines

Single Column

Forms should generally be used in a single column layout view. We have demonstrated a few different ways to do this below. If a multi-column layout is absolutely necessary then it should collapse and turn into a single column layout on mobile/tablet.

» See the Bootstrap documentation for more info


Labels and Placeholder Text

Always use labels and where necessary use placeholder text to provide additional info.


Standard Widths

By default input, select, and textarea are 100% width of the parent in Bootstrap. In order to control the sizing of input elements use the following grid classes only:

  • col-md-6: use for fields such as name, email, address, city, etc.
  • col-md-3: use for fields such as state, postal code, etc.
  • col-md-1: use for number inputs such as width, font-size, scroll pause, etc.
  • col-xs-6: pair with col-md-3 or col-md-1 only.

The row & col- grid class wrapper should be around the input element only. This prevents labels from being cut off.


Inline Forms

In the special case where you have to use an inline form, you'll have to set widths on the form controls used within. Using form-inline is an easy way to achieve this.

jQuery/AngularJS plugins are not standard HTML input elements. Therefore, you will have to specify a px / percent width in the CSS. Or use grid classes such as .col-xs-, .col-sm-, .col-md- & .col-lg-

Note: For viewports less than 768px wide form-inline will automatically collapse to a vertically stacked form.


Standard HTML Form Controls

Use of standard form controls instead of jQuery/AngularJS plugins is recommended. This allows the user to have a familiar native UI. Also, a lot of custom plugins are not optimized for mobile/tablet.

Checkboxes & Radio

Always use vertically stacked checkboxes and radio buttons


Validation States

Bootstrap includes validation styles for error, warning, and success states on form controls. Use .has-warning, .has-error, or .has-success classes for this. Additionally you can provide a glyphicon for feedback too.

Validation Feedback

For validation feedback please use Bootstrap contextual classes and alerts.
